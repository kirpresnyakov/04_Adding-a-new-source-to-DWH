Описание проекта

Цель: интеграция нового источника данных (external_source) в корпоративное хранилище данных (DWH) и построение инкрементальной витрины для анализа активности заказчиков.

Реализованные задачи:

    Изучение нового источника — анализ структуры и содержания таблиц craft_products_orders и customers в схеме external_source.

    Загрузка данных в DWH — написан скрипт loading_data_into_dwh.sql, который переносит данные из нового источника в схему dwh с распределением по таблицам измерений (d_customer, d_product, d_craftsman) и фактов (f_order). Загрузка поддерживает инкрементальность для дальнейших обновлений.

    Создание витрины dwh.customer_report_datamart — DDL-скрипт (create_datamart.sql) определяет структуру инкрементальной витрины с полями:

        идентификатор и ФИО заказчика

        потраченная сумма и доход платформы (10%)

        количество заказов, средний чек, медианное время выполнения

        самая популярная категория и самый популярный мастер

        статусная аналитика по заказам (созданы, в процессе, в доставке, завершены, незавершены)

        отчётный период (год/месяц)

    Инкрементальный расчёт витрины — скрипт обновления использует логику, аналогичную витрине мастеров, с применением оконных функций (ROW_NUMBER() для выбора одного мастера при равном количестве заказов).

Результат: новый источник успешно интегрирован, витрина готова к использованию в программе лояльности и аналитике поведения покупателей.

Файлы проекта:

    src/scripts/check_source_data.sql — проверка наличия данных в новом источнике

    src/scripts/loading_data_into_dwh.sql — загрузка данных из external_source в DWH

    src/scripts/create_datamart.sql — создание витрины customer_report_datamart
